本项目为 [openclaw](https://github.com/openclaw/openclaw) 最小实践，将按照以下需求逐步开发。P0 范围：仅 CLI 配置与本地对话；Telegram 真实对话、定时任务与 MCP 的具体实现属后续迭代。

以下所有命令行交互，命令行格式均为类似 `tclaw <command>` 的形式；

P0

1.  命令行交互
    1.1 可通过命令行进行配置，命令
    `tclaw config`
    进入配置，用户可以选择配置的内容（单选）；

        1.1.1 可通过命令行配置 TELEGRAM BOT 的相关配置，例如TELEGRAM_BOT_TOKEN，仅支持Telegram 【P0 暂时不需要实现真实的 Telegram 对话】；

        1.1.2 可通过命令行配置 OPENAI 的相关配置，例如OPENAI_API_KEY，仅支持 openai api 模型；

        命令行配置每次输入一项，例如
        ```bash
        tclaw config
        Choose the configuration you want to configure:
        [1] TELEGRAM BOT
        [2] OPENAI
        Enter the number of the configuration you want to configure:
        ```

        ```bash
        Choose the openai configuration you want to configure:
        [1] ALL
        [2] API KEY (Required)
        [3] BASE URL
        [4] MODEL NAME
        Enter the number of the configuration you want to configure:
        ```

        如果已经配置好的项，要在末尾加上 `[Configured]` 标记。如果是ALL，那么按顺序一个一个配置；

        ```bash
        TELEGRAM_BOT_TOKEN, Enter the value
        ```

        1.1.3 可通过命令行查看当前的所有配置（列出所有配置项 key 及脱敏后的值）

        ```
        tclaw config list
        ```

    1.2 可通过命令行进行对话，命令
    `tclaw chat`
    进入对话界面，用户可以输入对话内容；

        1.2.1 通过对话 `/exit` 退出对话；

        1.2.2 通过对话 `/summarize` 总结对话内容，并保存到日志文件中；

        1.2.3 通过对话 `/clear` 清空所有历史与记忆（chat.md 与 memory.md），需二次确认（输入 yes 或 y）后执行，相当于开启全新对话；

        1.2.4 用户退出对话后，在既有记忆摘要基础上增量合并本轮对话，生成新摘要并写入 `~/.tclaw/memory.md`；

        1.2.5 用户重启对话后，读取 `~/.tclaw/chat.md` 显示对话内容；展示时以上下文形式仅加载 `~/.tclaw/memory.md` 中的记忆摘要，完整历史仍存于 chat.md；调用 LLM 时显示「Thinking」等进度反馈，避免用户误以为程序卡顿；

2.  配置

    2.1 用户的所有配置存储在 `~/.tclaw` 目录下；首次使用时自动创建该目录及下述文件。

    2.2 用户的历史对话均存储在 `~/.tclaw/chat.md` 文件中；

    2.3 用户的对话历史记忆（总结后，每次对话都会重新调用）均存储在 `~/.tclaw/memory.md` 文件中；

    2.4 agent 学习的 skill 存储在 `~/.tclaw/skill.md` 文件中；

    2.5 agent 的人格定义存储在 `~/.tclaw/personality.md` 文件中；

    2.6 用户的配置（例如 TELEGRAM_BOT_TOKEN, OPENAI_API_KEY）均存储在 `~/.tclaw/config.json` 文件中；
