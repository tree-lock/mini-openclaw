# mini-openclaw 依赖文档

本文档说明项目需要安装的 npm 包，基于 [设计文档](设计文档.md) 与 [架构文档](架构文档.md) 的 P0 能力，并参考 [openclaw](https://github.com/openclaw/openclaw) 的依赖方案。项目使用 **Bun** 启动，Bun 已内置的能力不列入安装依赖。

## 1. 依赖总览

| 包名 | 用途 | 类型 | 与 openclaw 对应 |
|------|------|------|------------------|
| commander | CLI 子命令解析（`tclaw config` / `tclaw chat`） | 生产 | 同用 commander |
| @clack/prompts | 交互式配置（单选、输入配置值） | 生产 | 同用 @clack/prompts |
| chalk | 终端彩色输出（列表、提示、错误） | 生产 | 同用 chalk |
| openai | OpenAI API 调用（对话、总结） | 生产 | openclaw 用 undici 自拼请求；此处用官方 SDK 更简洁 |

**开发依赖**（已存在或建议保留）：

| 包名 | 用途 | 类型 |
|------|------|------|
| @biomejs/biome | 代码质量与格式化 | 开发 |
| @types/bun | Bun 类型定义 | 开发 |
| typescript | TS 类型检查 | 可选 peer |

## 2. 生产依赖说明

### 2.1 commander

- **用途**：解析 `tclaw <command>` 及子命令（如 `config`、`config list`、`chat`），分发到对应处理逻辑。
- **openclaw**：使用 `commander`（^14.0.3）作为 CLI 入口。
- **安装**：`bun add commander`

### 2.2 @clack/prompts

- **用途**：配置流程中的交互——单选（TELEGRAM BOT / OPENAI）、OPENAI 子菜单（ALL / API KEY / BASE URL / MODEL NAME）、逐项输入配置值（如 TELEGRAM_BOT_TOKEN、OPENAI_API_KEY、OPENAI_API_BASE、OPENAI_MODEL_NAME）。
- **openclaw**：使用 `@clack/prompts`（^1.1.0）做 TUI 交互。
- **安装**：`bun add @clack/prompts`

### 2.3 chalk

- **用途**：终端输出着色（如 `tclaw config list` 的列表、错误与提示信息），提升可读性。
- **openclaw**：使用 `chalk`（^5.6.2）。
- **安装**：`bun add chalk`

### 2.4 openai

- **用途**：P0 仅支持 OpenAI API；用于对话补全与退出时的总结调用。使用官方 SDK 可减少手写 HTTP 与请求体格式。
- **openclaw**：未直接依赖 `openai` 包，而是通过 `undici` 等发 HTTP 请求；本项目用 Bun 内置 fetch 也可，但引入 `openai` 更利于类型与后续扩展。
- **安装**：`bun add openai`

## 3. 由 Bun 提供、无需安装的依赖

以下能力由 Bun 或 Node 兼容层提供，**不要**再安装对应 npm 包：

| 能力 | 说明 |
|------|------|
| 环境变量 | `process.env` / `Bun.env`，Bun 会自动加载 `.env`，无需 dotenv |
| HTTP 请求 | 内置 `fetch`，调用 OpenAI API 时若不使用 openai SDK 也可直接用 fetch |
| 文件系统 | `Bun.file()`、`import fs from "node:fs"`、`import path from "node:path"` |
| 用户目录 | `import { homedir } from "node:os"`，得到 `~` 并拼接 `~/.tclaw` |
| JSON 解析/序列化 | `JSON.parse` / `JSON.stringify`，读写 config.json 无需额外库 |
| 逐行读取用户输入 | `import * as readline from "node:readline"` 或 Bun 的 stdin 读取，无需 inquirer 等 |
| 类型与构建 | TypeScript 与 Bun 内置运行，仅需 @types/bun 与可选 typescript |

因此无需安装：`dotenv`、`undici`、`inquirer`、`node-fetch` 等用于环境变量、HTTP、交互式问答的库（仅就 P0 而言）。

**P1 命令执行**：在对话中执行系统命令（如「帮我看看桌面有什么文件」→ 执行 `ls -l ~/Desktop`）使用 Bun 的 `Bun.spawn()` 或 Node 的 `node:child_process`，无需新增 npm 依赖。安全策略（黑名单、超时、输出截断）在应用层实现；若未来为审批流程或策略配置引入新依赖，再在本文档补充。

**P1 加载 skill**：从 `~/.tclaw/skills/*.md` 目录中读取 skill 文档，仅使用 Node/Bun 自带的 `fs` / `path` 处理文件与路径，YAML frontmatter 采用轻量级字符串解析，不引入额外依赖。

## 4. 安装命令

在项目根目录执行：

```bash
bun add commander @clack/prompts chalk openai
```

开发依赖已包含在 [package.json](../package.json) 中（如 Biome、@types/bun），无需额外安装；若尚未安装开发依赖，执行：

```bash
bun install
```

## 5. 与 openclaw 的差异说明

- **openclaw** 使用 `undici` 发 HTTP、未用 `openai` 包；本项目 P0 仅对接 OpenAI，用 `openai` 更省事，且 Bun 自带 fetch，不依赖 undici。
- **openclaw** 使用 `dotenv` 加载 .env；Bun 自动支持 .env，故不引入 dotenv。
- **openclaw** 依赖较多（Telegram、Discord、Slack、多模型等）；mini-openclaw 仅保留 P0 所需：CLI（commander + @clack/prompts）、终端样式（chalk）、OpenAI（openai）。
- 后续若增加 Telegram、定时任务、MCP 等，再按需引入对应包（如 grammy、croner 等），届时可参考 openclaw 的 [package.json](https://github.com/openclaw/openclaw/blob/main/package.json) 做选型。

## 6. 版本与兼容性

- 建议使用 Bun 1.x 运行；依赖均支持 ESM（`"type": "module"`）。
- 具体版本以 `bun add` 解析结果为准；若需锁定，可使用 `bun.lockb` 或与团队约定版本范围。
